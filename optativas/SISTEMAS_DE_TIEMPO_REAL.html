10. Justificación técnica

En muchos sistemas electrónicos el tiempo es parte de la corrección del sistema. En un sistema inestable que emplea un sistema de control para estabilizarlo no nos podemos permitir que el sistema de control no tenga tiempo de hacer su trabajo. La consecuencia puede ser letal, una explosión, la caída de un helicóptero, una máquina herramienta fuera de control.
No se trata de buscar algoritmos más rápidos, sino de diseñarlo de manera que, aún en el caso peor, el sistema tenga tiempo de hacer lo que debe. La planificación y el análisis de planificabilidad de estos sistemas constituye la parte central del curso. Es fácil anticipar que planificar implica reservar recursos que no se van a utilizar normalmente. A veces demasiados recursos… Se verán diversas estrategias para mejorar significativamente la utilización del sistema. Algunas técnicas, como la programación dirigida por interrupciones, deberán reconsiderarse con mucho cuidado y se deberá rediseñar la estrategia de reacción ante eventos esporádicos del sistema. Lenguajes como Python, que disponen de gestión automática de memoria dinámica, no podrán ser empleados.
Las prácticas se realizarán sobre un sistema empotrado que ejecuta un sistema operativo de tiempo real estricto. Los programas se realizarán en C o en C++.
